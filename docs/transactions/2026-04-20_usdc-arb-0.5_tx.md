4. **First attempt** (`call_OcF3Pazz0xWWnZkhOJvltbSp`) failed server-side simulation — delegation scope/execution path mismatch

## Notes / takeaways

- First-attempt simulation failure is recoverable by a follow-up prompt within the same chat; CoinFello re-issues a corrected delegation.
- Each retry is a **separate signed delegation** (new call ID) — user must re-approve. Scope must be re-verified each time.
- Smart account `0xD9a4...9717` remains in user custody; Secure Enclave key is never exported.
- No gas was paid by the EOA; the smart account + delegate handled the transaction.
